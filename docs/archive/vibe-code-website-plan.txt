A. FINALNA REKOMENDACJA STACKU
Rekomendacja: VitePress (Node.js) Uzasadnienie Wyboru: Operator aktualnie używa narzędzia Vite do kompilacji paczek aplikacji (Bouwplaats Chaos / zzpackage). Wprowadzenie VitePress korzysta z tych samych globalnych uwarunkowań pamięci mięśniowej. Posiada on jeden z najlepszych na rynku wbudowanych systematycznie trybów "Dark Premium", natywnie wspiera najszerszą gamę powiadomień Markdown (::: danger / ::: warning), ładuje się błyskawicznie jako Single Page Application i pozwala na kompilację SSG bez żadnych wyłamań architektonicznych. Konfiguracja nawigacji sprowadza się do tablic w pojedynczym pliku config.js.

G. MINIMALNE WYMAGANIA WIZUALNE
Styl uderzeniowy Dark Premium Command Center:

Paleta: Grafitowe czernie tła (#0A0A0A do #111111). Tekst o obniżonym kontraście (#E5E7EB - nie oślepiający białym).
Typografia: Inter (Primary bezszeryfowo) & Jetbrains Mono / Fira Code z jaskrawo podświetloną syntaksą szmaragdową / fioletową.
Kształty: Zero nadmuchanych marginów i paddingów. Styl militarno-deweloperski bazujący na ostrych krawędziach.
Zróżnicowane Markdown Admonitions (VitePress ma to wbudowane):
::: danger (Dla zabójczych antywzorców).
::: warning (Dla kwarantanny - wymagany backup produkcji).
::: info (Dla rozdzielenia kontekstowego agenta).
H. SKRYPTY POMOCNICZE POWERSHELL (Dla Win)
W katalogu /scripts przygotowane stricte pod wariant Windowsowy systemu Dowództwa.

Start-Cockpit.ps1
Cel: Start środowiska. Odpalasz w VSCode (PowerShell) komendą. Skrypt przechodzi do głównego katalogu, uruchamia npm run docs:dev uwalniając wątek VitePress'a, budzi instancję do local-hostu i wstrzymuje wylogowanie z logami operacyjnymi na Twoim oknie.
New-Log.ps1
Parametry: New-Log.ps1 -Type "incident" (lub "session", etc.).
Cel: Skrypt kopiuje plik np. /templates/tmpl-incident.md i zapisuje w folderze /journal/ jako RRRR-MM-DD_[time]_incident.md, natychmiastowo otwierając wbudowanym edytorem VS Code ten skopiowany plik w okienku. Zero klikania kursorem, sama egzekucja myśli.
I. PLAN ETAPOWEJ BUDOWY PRZEZ GEMINI CLI
Jako architekt narzucam harmonogram dla Geminii CLI:

Krok 1 (Setup): CLI instaluje npm install -D vitepress, konfiguruje package.json.
Krok 2 (Skeleton): CLI ignorując zawartość, generuje pełne sztywne drzewo folderów według schematu i rozmieszcza puste pliki z adnotacjami YAML dla Placeholderów.
Krok 3 (Szablony): CLI programuje i buduje struktury powtarzalne dla plików w /templates.
Krok 4 (Hardcoded Documents): CLI wciska logikę dla plików "Realna Treść Strukturalna", adaptując Global-Rules. CLI konstruuje zawartość Frontmatter.
Krok 5 (Konfiguracja SSG): CLI generuje config nav i sidebar w docs/.vitepress/config.mts i wymiata niebezpieczne style klasowego szumu ze zmiennych CSS.
Krok 6 (Skrypty): CLI rzeźbi PS1 dla sprawnego operowania systemem na platformie Windows.
